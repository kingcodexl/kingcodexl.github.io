区分：JVM内存模型及java内存模型

java虚拟机：hotspot

jvm规划，相当于接口，在里面定义必须实现的方法。

方法区：运行时常量池，虚拟机加载类的信息。——》java类的模板信息，为什么可以通过反射得到一个对象就是存在类的模板。——》所有线程共享

java堆区：对象有引用对象（局部变量）和真实对象（存在堆中）之分 ——》所有线程共享

java栈：线程私有

本地方法：native 线程私有

程序计数去： 方法执行到哪个地方——》线程私有

一个线程只要有调用方法就会有一个栈帧，

主要存：1. 局部变量表 2. 操作数栈 3.动态链接 4.出口记录

堆内存理解

其实线程独占的三部分gc基本不动，在线程共享的两部分方法区gc也是基本动不了。那gc基本唯一会经常操作的就是堆内存。堆内存的大致划分：
JDK 1.8和以前

JDK 1.8以后 不在用永久带，而是Meta Space已经不在堆内存中。

1.1 新生代
所有new的对象都在堆里面。对象优先分配到Eden区，可以通过参数-XX：SurviviorRatio=8来指定Eden区和Survivior的比例，这样划分的依据和好处是根据gc的回收算法来定的。做到了内存利用率高。

经过几次(可以通过参数设置)minor gc还存活的对象进入S0，再经过几次minor gc还存活进入S1，达到minor gc的阈值（-XX:MaxTenuringThreshold=？）进入老年代。

还有就是动态对象年龄判定，相同年龄所有对象的大小总和 > survivor空间的一半，直接晋级老年代，不需要进行年龄阈值的判断。

这个区域的理想是98%以上的对象能够被回收。

1.2 老年代
大对象（-XX:PretenureSizeThreadshold=1024）直接进入老年代，在新生代里面长期存活的对象进入老年代。老年代对应的是major gc。

1.3 永久带
包含元数据信息，如class，method的detail信息

二、空间分配担保（就像贷款一样） -XX:HandlerPromotionFailure
Minor GC之前检查
老年代最大可用连续空间是否 > 新生代所有对象总空间。
大于—-> Minor GC
不大于—>老年代最大可用连续空间是否>历次晋升到老年代对象的平均大小
大于—–> Minor GC
不大于 —–> Full GC
-XX:HandlerPromotionFailure
注：Minor GC 是针对新生代的垃圾回收，MajorGC是老年代的，Full GC 是Minor GC 加上 MajorGC

三、对象的创建过程（new的过程）

3.1 怎么在堆中分配内存
a,堆规整的前提下——–> 用指针碰撞
b,堆不规整的 ———> 用空闲列表
堆规整不规整是由垃圾回收器的策略决定的
3.2 分配内存中线程安全的问题
比如多个线程同时new对象，那指针的位置归谁？
解决办法 1）同步加锁 —效率低
2） 在堆中给每个线程分配区域。
3.3 对象的结构
a, Header:自身运行的数据(很多)
b, InstanceDate 程序员写的
c, Padding
3.4 对象的访问定位
a,使用指针 hotspot 采用
b,使用句柄 （多一步操作，在堆中有句柄池。优点是栈中的地址不会变）

内存调优

Gc 流程

  • 当eden满了，触发young GC；
  • young GC做2件事：一，去掉一部分没用的object；二，把老的还被引用的object发到survior里面，等下几次GC以后，survivor再放到old里面。
  • 当old满了，触发full GC。full GC很消耗内存，把old，young里面大部分垃圾回收掉。这个时候用户线程都会被block。

JVM调优哪些部分：

  1. 减少父GC频率
  2. 减少这个过程时间

不能让程序轻易执行父GC

什么情况下回Full GC，system.GC，FuuGC满了。

JVM内存中最大的一块，是所有线程共享的区域，在JVM启动时创建，唯一目的就是用来存储对象实例的，也被称为GC堆，因为这是垃圾收集器

管理的主要区域。Java堆还可分为：新生代和老年代，其中新生代还可再分为：Eden：From Survivor：To Survivor = 8:1:1，有面试官会问

为什么8:1:1，可参考：垃圾收集算法——标记-清除算法、复制算法、标记-整理算法、分代收集算法 中的复制算法

String str = "abc" + "xyz"

创建了几个对象。

首先明确对象和引用对象的区别。对象是真实占有内容内存的，而引用对象只有一个引用地址。

所以可以这样说：一个str引用对象，还有一个字符串对象

  1. str 引用对象
  2. abc xyz abcxyz

一共四个对象。这是JVM1.6之前的

只产生了一个对象，还有一个str引用对象。JVM读到这行代码，常量是会被优化的，字面量会直接编译器优化，加号去掉变成字符串常量。

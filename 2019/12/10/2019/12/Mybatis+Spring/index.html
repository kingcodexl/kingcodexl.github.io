Mybatis如何与Spring整合在一起的
视频笔记 Mybatis
服务端 数据库
发布日期:   2019-12-10
文章字数:   1.1k
阅读时长:   4 分

默认情况下，MyBatis单个如何使用，其次MyBatis Spring如何工作的。

单个MyBatis

如果是单单是MyBatis，要解决的问题就是如何通过接口生成对象，而且还能执行接口方法上写的sql。

首先明确一点，注解和XML作用是一样的，都可以看做是配置信息。XML可以domj解析，注解通过反射解析，最终都可以拿到配置信息，所以差别不大。

更重要的是解决对象如何创建的问题。

解决对象创建

接口如何转为对象，很明显的要用到动态代理。动态代理可以把一个接口变成对象。核心就是需要实现一个Handler类，用于实现接口定义的方法。

现在解决对象及sql问题。

总结一下

  1. 对象是动态代理创建，不是new，
  2. 希望这个动态代理自己创建，并且结合Spring容器

两个前提，我们可以把一个类给Spring容器管理，也可以把一个对象拿给容器管理。不同点是把一个类给容器管理，那么就不需要关心创建的问题；把对象拿给容器管理@Bean，需要自己管理对象的创建。

因为动态代理对象是我们创建的，所以不能使用注解的方式，可以直接解决的就是用FactoryBean.现在问题就是如何将我们产生的代理对象放到Spring中。注意这里是对象而不是类。

如何将自己产生的对象放到Spring中

注意这里是对象，不是类。用@Service，@Compent是把类交给Spring

这里用FactoryBean接口实现。返回具体的对象及其类型。

@Component
public class WeslyFactoryBean implements FactoryBean {

    @Override
    public boolean isSingleton() {
        return true;
    }

    @Override
    public Class<?> getObjectType() {
       return UserMapper.class;
    }

    @Override
    public Object getObject() throws Exception {
        UserMapper userMapper = (UserMapper) WeslySession.queryMapper(UserMapper.class);
        return userMapper;
    }
}

这样就可以把动态代理生成的对象放到Spring当中。进一步的，我们还可以把返回的类型参数化，也就是把UserMapper作为参数传入进来，这样就可以扩展到其他Mapp了。其实就相当于给Bean属性赋值的逻辑，可以使用xml作为配置。

<bean id="usermapper" class="com.xx.xx.xx.WeslyFactoryBean">
    <property name="mapper" value="com.xxx.xxx.UserMapper" />
</bean>

这样每一个Mapper都需要创建一个新的Xml配置。

为了不每一次都配置一个，那么就需要一次把多个mapper注入到容器。需要通过Spring扩展来实现，具体来说就是自定义BeanDefine，因为自定义BeanDefine，可以给BeanDefine设置属性，这样就避免了@Service方式不能设置属性的问题。

Spring扩展

上面已经实现了将代理对象放到Spring中。但是还不够智能，一次智能放一个Mapper进去，所以还需要改进。上面我们是通过注解的方式，自动装配了WeslyFactoryBean，然后在WeslyFactoryBean中将定义的Mapper放到容器里面。

为了扩展性，我们希望把这里的UserMapper参数化，作为属性。比如

public class WeslyFactoryBean implements FactoryBean {
    private Class mapperInterface;
    @Override
    public boolean isSingleton() {
        return true;
    }

    @Override
    public Class<?> getObjectType() {
       return mapperInterface;
    }

    @Override
    public Object getObject() throws Exception {
        Object o = WeslySession.queryMapper(mapperInterface);
        return o;
    }

    public Class getMapperInterface() {
        return mapperInterface;
    }

    public void setMapperInterface(Class mapperInterface) {
        this.mapperInterface = mapperInterface;
    }
}

这样虽然抽象出了属性，但是属性如何注入呢。上一步我们用@Component，但是做不到填充属性。为了解决这个问题就需要用到ImportBeanDefinitionRegistrar。

ImportBeanDefinitionRegistrar提供了自定义BeanDefine，也就是我们可以设置注入Bean的属性值。这样就解决了不能注入属性的问题。

自动注入属性，

public class WeslyImportBeanDefineRegistart implements ImportBeanDefinitionRegistrar {
    @Override
    public void registerBeanDefinitions(AnnotationMetadata importingClassMetadata, BeanDefinitionRegistry registry) {
        BeanDefinitionBuilder beanDefinitionBuilder = BeanDefinitionBuilder.genericBeanDefinition(WeslyFactoryBean.class);
        AbstractBeanDefinition abstractBeanDefinition = beanDefinitionBuilder.getBeanDefinition();
        //添加属性值这样就可以避免@Service
        abstractBeanDefinition.getPropertyValues().add("mapper","test.com.mock.WeslyFactoryBean");
        registry.registerBeanDefinition("xxx",abstractBeanDefinition);
    }
}

剩下的就是如果将WeslyImportBeanDefineRegistart放到容器里面，这个就很简单了。因为他不用设置什么属性。直接用@Service也是可以的，但是为了区分作用，还是用@Import(xxx.class)包装一下，这样就和Mybatis一样了。在Appconfig类中加上这个自定义的注解即可。

@Retention(RetentionPolicy.RUNTIME)
@Import(WeslyImportBeanDefineRegistart.class)
public @interface WeslyScan {
}

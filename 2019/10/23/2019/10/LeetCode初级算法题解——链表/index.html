1. 删除链表中的节点

原题

请编写一个函数，使其可以删除某个链表中给定的（非末尾）节点，你将只被给定要求被删除的节点。

现有一个链表 -- head = [4,5,1,9]，它可以表示为:


示例 1:

输入: head = [4,5,1,9], node = 5
输出: [4,1,9]
解释: 给定你链表中值为 5 的第二个节点，那么在调用了你的函数之后，该链表应变为 4 -> 1 -> 9.
示例 2:

输入: head = [4,5,1,9], node = 1
输出: [4,5,9]
解释: 给定你链表中值为 1 的第三个节点，那么在调用了你的函数之后，该链表应变为 4 -> 5 -> 9.


说明:

链表至少包含两个节点。
链表中所有节点的值都是唯一的。
给定的节点为非末尾节点并且一定是链表中的一个有效节点。
不要从你的函数中返回任何结果。

分析

这道题算是链表最简单的一题了吧。要删除传入的节点，因为已经给了节点，我们就不用查找了。常规的删除节点方法是，将前一个节点指向后一个节点。但是这里我们不知道前一个节点是什么，很多同学就懵逼了

不能删除当前节点，但是我们可以修改当前节点呀，题目并不是要你一定要删除这个节点的内存地址，而是只要满足条件即可。我们把后一个节点的值赋给当前节点，然后把当前节点指向下一个的下一个节点就可以了。

有些同学想当然的直接用node = node.next。为什么这样的写法是错的。

这涉及到函数传参值引用和地址引用的问题。基本上所有的编程语言，参数传递都是值传递，除非手动设置为地址传递，也就是默认传递的是当前值的一个副本。所以改变参数node的地址是改变不了原来链表中node的地址。

覆盖解法

def deleteNode(self, node):
        """
        :type node: ListNode
        :rtype: void Do not return anything, modify node in-place instead.
        """
        node.val = node.next.val
        node.next = node.next.next

2. 删除链表的倒数第N个节点

原题

给定一个链表，删除链表的倒数第 n 个节点，并且返回链表的头结点。

示例：

给定一个链表: 1->2->3->4->5, 和 n = 2.

当删除了倒数第二个节点后，链表变为 1->2->3->5.
说明：

给定的 n 保证是有效的。

进阶：

你能尝试使用一趟扫描实现吗？

分析

如果没有限制扫描次数，很直观的思路就是先扫描一次链表，得到链表长度。然后根据输入的n得到要删除节点的位置。接下来进行第二次扫描，找到位置之后删除。

首先我们将添加一个哑结点作为辅助，该结点位于列表头部。哑结点用来简化某些极端情况，例如列表中只含有一个结点，或需要删除列表的头部。这点是困扰了很久，类似于几何题中，添加一条辅助线的感觉。。

除此之外有一个非常巧妙的办法，用两个指针，第一个真真先走n步，然后第二个指针和第一个指针在同时走。当第一个指针走到末尾的时候说明第二个指针就走到了倒数第n个的位置。可以把这种解法理解为先后指针，和快慢指针有些类似。

两次循环哑结点解法

def removeNthFromEnd(self, head: ListNode, n: int) -> ListNode:
        count = 0
        tmp = head
        #得到长度
        while tmp != None:
            count +=1
            tmp = tmp.next
        #得到删除的索引位置
        index = count - n
        helper_node = ListNode(1)
        helper_node.next = head
        #辅助节点
        tmp = helper_node
        while index > 0:
            index -= 1
            tmp = tmp.next
        #找到节点记性删除
        tmp.next = tmp.next.next
        return helper_node.next

前后指针一次遍历解法

 def removeNthFromEnd(self, head, n):
        """
        :type head: ListNode
        :type n: int
        :rtype: ListNode
        """
        helper_node = ListNode(1)
        helper_node.next = head

        node = helper_node
        count = n
        #移动n次
        while count != 0:
            count -= 1
            node = node.next

        #辅助节点，便于删除
        pre_node = helper_node
        #找到删除的位置
        while node.next != None:
            pre_node = pre_node.next
            node = node.next

        pre_node.next = pre_node.next.next
        return helper_node.next

3. 反转链表

原题

反转一个单链表。

示例:

输入: 1->2->3->4->5->NULL
输出: 5->4->3->2->1->NULL
进阶:
你可以迭代或递归地反转链表。你能否用两种方法解决这道题？

分析

翻转链表最常见的就是原地反转。需要定义一个辅助节点来保存当前的下一个节点。思路并不复杂。涉及到的操作就是，移动修改指向。

基于这种思路，还有一些其他类似的方法。我们还可以添加一个辅助节点它的下一个就是当前头节点，每循环一次，就把该节点放到辅助节点后面并且把之前的节点连接到该节点之后。最后返回辅助节点的下一个节点即可。

原地反转解法

    def reverseList(self, head):
        pre = None
        current = head
        while current != None:
            node = current.next
            current.next = pre
            pre = current
            current = node
        return pre

辅助头结点解法

    def reverseList(self, head):    
        pre = ListNode(1)
        current = head.next
        pre.next = head
        while current!= None:
            node = current.next
            current.next = pre.next
            pre.next = current
            current = node
        head.next = None
        return pre.next

4. 合并两个有序链表

原题

将两个有序链表合并为一个新的有序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。 

示例：

输入：1->2->4, 1->3->4
输出：1->1->2->3->4->4

分析

思路比较简单，就是不断比较，每次移动其中其中一个链表的节点。注意遍历完之后可能还有多余部分，需要把多余的部分添加到末尾。

新节点

def mergeTwoLists(self, l1, l2):
        """
        :type l1: ListNode
        :type l2: ListNode
        :rtype: ListNode
        """ 
        head_node = ListNode(1)
        head = head_node
        while l1 != None and l2 != None:
            if l1.val <= l2.val:
                head_node.next = l1
                l1 = l1.next
            else:
                head_node.next = l2
                l2 = l2.next
            head_node = head_node.next
        if l1 != None and l2 == None:
            head_node.next = l1
        if l2 != None and l1 == None:
            head_node.next = l2
        return head.next

5. 回文链表

原题

请判断一个链表是否为回文链表。

示例 1:

输入: 1->2
输出: false
示例 2:

输入: 1->2->2->1
输出: true
进阶：
你能否用 O(n) 时间复杂度和 O(1) 空间复杂度解决此题？

分析

如果不考虑时间复杂度、空间复杂度。思路就很简单了。遍历链表，同时取出各个几点的值，保存到list中。然后判断list是否是左右对称即可。

还有一种取巧的方式可以满足$O(1)$的空间复杂度。用快慢指针，一个指针每次走一步，一个指针一次走两步。当第二个指针走到末尾的时候，第一个指针刚好走到一半的位置。然后翻转前一半链表。进而和后一步进行比较。

其实也可以不用快慢指针，可以先得到链表的长度，然后除2得到一半的位置。然后在翻转后一半（当然前一半也是可以的），思路就和快慢指针一样了。问题的关键在于如何得到链表的中心点，得到中心点之后然后翻转各个部分，进而比较。

快慢指针在写代码的时候有个隐藏问题，因为事先不知道链表个数是偶数还是奇数个。所以在翻转的时候需要统一规则。比如奇数1->2->1，fast最后在右边1的位置，slow在2的位置，为了比较的方便，需要将slow往后挪动一个位置。因为是翻转后一半，需要把中间位置去掉。

转为list判断解法

def isPalindrome(self, head):
        """
        :type head: ListNode
        :rtype: bool
        """
        if head == None:
            return True
        l = []
        while head != None:
            l.append(head.val)
            head = head.next
        for i in range(len(l) // 2):
            if l[i] != l[len(l) - 1 - i]:
                return False
        return True

快慢指针

  def isPalindrome(self, head):
        """
        :type head: ListNode
        :rtype: bool
        """
        if head == None or head.next == None:
            return True

        #找到中间节点
        slow,fast = head,head
        while fast != None and fast.next != None:
            slow = slow.next
            fast = fast.next.next
        # 奇数长，fast指针在最后一个，slow在最中间，slow需要往后过一个
        # 偶数长，fast为空，slow指针中点过一个
        if fast != None:
            slow = slow.next
        #翻转前半部分
        pre = None
        current = slow
        while current != None:
            node = current.next
            current.next = pre
            pre = current
            current = node
        #比较是否相等
        while pre != None and head != None:
            if pre.val == head.val:
                pre = pre.next
                head = head.next
            else:
                return False
        return True

6. 环形链表

原题

给定一个链表，判断链表中是否有环。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。



示例 1：

输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。


示例 2：

输入：head = [1,2], pos = 0
输出：true
解释：链表中有一个环，其尾部连接到第一个节点。


示例 3：

输入：head = [1], pos = -1
输出：false
解释：链表中没有环。



进阶：

你能用 O(1)（即，常量）内存解决此问题吗？

分析

一个简单的思路记录一下当前遍历过的节点，如果这个节点之前出现过，那么就肯定有环。

另一种思路就比较取巧了。用一对快慢指针，如果没有环，那么快指针肯定比慢指针先到末尾，如果有环，那么快慢指针肯定会碰头。如何理解必定相遇，就如同两个人绕着圆圈跑步，a速度就是比b快。那么无论是a在b前，还是a在b后，因为是圆形，最终肯定能追到b

需要注意如果链表元素不足两个那么就不可能存在环，需要提前判断。

记录是否出现过

def hasCycle(self, head):
        """
        :type head: ListNode
        :rtype: bool
        """
        node = head
        nums = []
        while node != None:
            if node not in nums:
                nums.append(node)
            else:
                return True
            node = node.next
        return False

快慢指针解法

def hasCycle(self, head):
        """
        :type head: ListNode
        :rtype: bool
        """
        if head == None or head.next == None:
            return False
        slow = fast = head
        while fast != None and fast.next != None:
            slow = slow.next
            fast = fast.next.next
            if slow == fast:
                return True
        return False

总结

链表基本的操作是移动和连接。

移动

while node != None:
    node = node.next

连接

node.next = node

基本上所有的链表题都是基于这两个操作。

比如翻转就要用到链接。但是如果改变了当前节点的下一个那么原来后面的元素就无法取到。所以一般在进行改变当前下一个节点的时候定义一个临时的节点，保存当前下一个节点。这样就可以找到之后节点了。

因为链表就这些节点，有时候才操作时非常不方面，很多情况下我们都会定义辅助节点来简化逻辑。比如在删除倒数第n个节点这题中。定义一个辅助接点，它的下一个就是当前头结点，这样就解决了元素只有一个的情况。

快慢指针是常见的技巧，通常一共来定位。因为链表不能随意访问元素，只能一次一次遍历，所以用快慢指针来确定元素的索引。比较经典的就是用快慢指针来检查链表是否有环、删除倒数第n个元素。

但是在使用快慢指针的时候，需要非常注意结束的位置。形式如下：

while fast != None and fast.next != None:
            slow = slow.next
            fast = fast.next.next

如上代码慢指针移动一次，快指针移动两次。因为不知道链表元素的个数，不知道快指针会停在什么地方。当不存在环的时候。

如果是偶数个，fast最终等于None。slow停在正中间过1个的位置

如果是奇数个，fast最终会停在最后一个节点。slow停在正中间的位置

这也是为什么在移动快指针的时候需要用fast != None and fast.next != None两个条件来限定。fast != None是处理偶数的情况，fast.next != None是处理奇数的情况。

# 风险管理

产品经理百分之七十的时间都是在沟通。沟通是软性能力。产品经理的差距就是在个人魅力及沟通管理能力。

风险管理是指如何在项目或者而企业一个肯定有风险的环境里，把风险可能造成的不良影响减至最低。

意义：完成既定的目标

风险管理的方法

风险识别

例子：

风险估测及风险评价

其中目标不明确是最常见的问题。

风险管理

一定要跟老板再沟通，确认。一切要有文字备案，有证据。目标也要告诉老板。

了解做这个事情的目的。

预期管理

  • 合理的背景分析
  • 理性的市场预估
  • 风险提前判断
  • 获取领导确认

过程风险管理

  • 合理工作量评估
  • 项目节点：这个非常重要，比如一个周，每天站立晨会
  • 需求优先级：
  • 减少重复需求

需求变更了怎么沟通

  • 明确原因？变更大不大，技术，市场，老板，还是产品的原因。
  • 技术：积极减少不必要的需求
  • 市场：向上反馈，获取支持后积极调整
  • 老板：明确必要性，适度调整
  • 产品：自我检讨，保障项目顺利进行。

了解变更之后的原因

风险发生后的管理

  • 根据情况可适当调整发布时间
  • 及时与关键岗位沟通——老板，运营，运维。各个负责的部门
  • 及时复盘总结

合作项目，做任何事情一定要明确，文档化。文档是由做事情的一方输出；外包项目前期沟通一定，一定要仔细沟通。

沟通管理

沟通的目标

  • 说明事物
  • 表达情感
  • 建立关系
  • 进行企图

沟通工具的使用

每种工具使用场景不通过。最有效的是面对面沟通，可以知道对方的情感。

需求阶段沟通的重点

  • 明确需求目标
  • 明确时间点、里程碑，有助于控制项目节奏
  • 明确产品方向

需求评审会就是沟通核心时机，理清上面的三个问题。

一些小技巧可以让各个角色更好的投入项目：

  • 需求讨论起，尝试与开发沟通可实现性
  • 交互时期，拟定交互信息，让交互生产原型
  • 评审期，反复沟通基本目标，确保团队投入。

开发阶段沟通重点

  • 了解每日进展，站立会
  • 明确开发难度，了解有什么困难，风险点
  • 减少需求变更
  • 多体验保证需求预期

发布阶段沟通重点

  • 运营支持沟通
  • 数据反馈沟通：数据，效果如何
  • 用户沟通

不同角色沟通方法

直属领导

老板对你说的话，尽量确认，最终达到共识。

为什么老板要经常问，因为老板没有安全感，多向老板反馈。

有机会就尽可能和老板面对面谈，混个脸熟。一个礼拜，一个月保证和老板说上话，让老板觉得自己有存在。

对于其他领导的要求，技巧如下：

一定要记得向上求助，不要直接答应好还是不好。核心让他觉得你在积极的帮他解决问题，不是你不想干，事情不要自己答应下来。

开发

多和B沟通，A这种不能直接发现问题。多鼓励A来给需求找问题。C这种就把需求写明白。D这种，先肯定他，暗示风险。

核心和技术达成产品的统一标准，也就是达成解决方案的共识。其次功劳是大家的。

测试

及时同步需求，有需求变更了一定要及时通知测试。

设计师

运营

这样做的目标是什么，更多是用目标导向来确定运营的需求。

合作需求方

比如和外包公司的关系

沟通结果一定要记录，把握好双方的预期，不要太大。

下级沟通

不要只跟他说结论，信息尽可能对称。

任何人都需要面子，表扬在人前，批评在人后。

建立有效沟通机制

在团队磨合中，需要注意：

  1. 尽可能统一判断的标准，比如一切以用户价值为依据
  2. 不要试图说服，而是找到折中点
  3. 充分理解每个角色的内心真实想法

复盘，真的很重要

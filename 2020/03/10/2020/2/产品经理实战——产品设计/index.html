互联网产品设计

互联网产品设计是指基于战略需求和用户需求，形成方案直至实现方案完成为止的一系列工作的准备和管理。

基本原则

产品设计基本原则：

  • 需求原则：用户导向
  • 信息原则：结合分析结果
  • 有效原则：解决了问题
  • 逻辑原则：体现在产品文档上，需要符合一些逻辑
  • 效率原则：用有限的资源做出效果

以用户为中心的产品设计

在不同的阶段都有不同的产品输出物。

产品设计

设计都是有套路的，也就是自己的方法总结。

人性法

人性的弱点就是可以被利用的部分。

贪婪例子：设计基本要点，突出免费，限量，高价值，真实，低投入高回报。

懒：外卖产品，设计要点，突出快捷，选择多，优惠券

色：交友产品，设计要点，颜值导向，匿名，地理位置，Sex幻想

第一步：挖掘人性最根本的需求源，懒，贪，色，傲

第二步：发散(不是单一的方式，可以用迂回的方式)思维寻找解决方案，直击人性弱点….

第三步：选择合适的方案快速试错，检验是否成利用弱点….

例子：如何设计一个可以提高用户阅读效率的产品or功能点？

分析出弱点：懒不要花时间找信息，贪不想被社回淘汰。方案可以是精选内容，垂直领域内容，语音播报，简报。

例子

市场上的同质化产品越来越多，满足用户的基本需求已经不足以留住用户，从“满足需求”到“让用户惊喜 ”，拼得是对人性的洞察。

不管是七宗罪还是马斯洛需求层次理论，都不足以囊括人性的全部，下面挑选几个人性关键词，从表现、代表产品（大）、具体的产品设计方法（小）几个方面详细说明。

映射法

网络秀场和传统秀场歌厅是有映射关系的。将线下的产品对应到线上之后应该怎么做。比如传统秀场有颜值导向，有金主，艺人与金主的互动。

网络买票和传统的售票厅具有映射关系，参考传统购买就知道有查询，支付，该退票，排队功能等。

第一步：对比现实场景，有什么是类似的需求..

第二步：类比线上环境，需求可以如何被满足…

第三步：选择合适的方案，提高需求满足效率及效果…

例子：互联网课堂对比线性课堂，可以有哪些产品设计？

线下课堂：授课环境，授课工具，互动方式，检查作业，纪律维护

线上课堂：虚拟课堂环境，PPT讲解工具，截图，音频，视频，画笔，签到，语音聊天，文字聊天等。

不能解决的问题，每个学生的学习效果。

参考法

微创新关键点：

  • 从小处着手，贴近用户需求心理
  • 快速出击，不断试错
  • 模仿并不一定是抄袭

QQ游戏大厅与模仿联众游戏大厅，QQ微创新点就是利用QQ作为游戏入口，导入大量的用户。

微信模仿Kik与米聊相似，微信创新点产品性能更优，扩展陌生人领域，结合原有优势导流。

唯品会模仿Vente_Privee，微创新点移植国外模式到中国，满足国人虚荣心，继而变成尾货市场。

第一步：模仿，确定需求…

第二步：列举，需求清单…

第三步：创新，更优的解决需求…

项目流程详解

产品设计涵盖全过程。

主要工作

需求阶段：完成需求从脑细胞到交互，原型，文档的过程。思想家，作家，画家。因为想不清楚就写不出来。

开发阶段：监督需求实现，排除万难保障需求的完成时间。外交家，建工。这个阶段同步思考下一阶段需要做什么。

发布阶段：完成需求检验及用户意见搜集，上线数据分析等工作，继而这里需求进入下一轮的产品设计。数据分析师，客服。产品是产品经理的孩子。

小结

产品设计：

  1. 关注需求
  2. 合理制定方法
  3. 明确迭代规划
  4. 合理展示架构
  5. 以用户为核心设计

多思考战略层的问题。

各个阶段的任务

整个过程是由抽象到具体的过程。

方案与规划

回顾

市场分析：确定是否有利可图

竞品分析：让可图的利益最大化

需求分析：需求不是特定的解决方式，是希望满足某种欲望或者需要。

在工作中大多数时候都是听到的是一种解决方案，但它大部分不是真实的需求。我们需要挖掘真实的需求，提出满足需求的最佳解决方案。

需求和方案的差异

需求：感性的，一个点，有限的，心灵的，用户的

方案：理性的，一个面，多样性，金钱的，企业的

互联网产品方案定义：是基于用户需求，并根据行业及市场背景，在有限时间内可按迭代完成的，有利于本企业建立竞争优势及获得更多用户并产生更多效益的一套产品策略及运营策略的方案组合。

产品方案的要求

设计方法

基于分析结论：市场&竞品，用户&需求——》形成解决方案：定位&目标，逻辑&功能。

产品方案是重思路，策略，逻辑，而轻具体实现细节。

基本内容

论据需要进行提炼，论据与最终提出的产品方案有直接练习，尽可能以数据的形式展示。

产品定位，愿景及目标的差异

  • 产品定位：产品是怎样满足目标消费者而活着目标消费市场的需求

  • 愿景：企业长期发展的方向目标理想愿望及企业自我设定的企业责任和义务

  • 产品目标：本次产品的退出能达到具体的怎样的效果，比如分别满足了用户及企业什么需求。

核心逻辑表达

核心逻辑：产品的核心模块，核心竞争力，差异化部分。只把关键点展示出来。

xxxx是xxxx。

项目时间规划

不同场景对应不同方案：

  1. 预研类方案的规划主要是打成大目标的整体时间规划。三年计划
  2. 具体产品方案的时间规划，主要是开发用时的基本评估。版本计划

描绘美好未来

产品方案的使用

  1. 个人思路的整理
  2. 深刻理解需求，提炼最合适的解决路径
  3. 项目工作使用
  4. 项目预研，项目申报，立项
  5. 甚至-BP

常见规划中容易产生的问题有：

  • 未论证需求真伪
  • 未讨论预期
  • 为合理规划投入
  • 需要对项目进行规划

意义在于

明确目标，有利于资源协调及执行，更有利于团队一起努力。

产品规划具体做法

立项

  • 怎么立项
  • 为什么要立项
  • 是不是小公司就不需要立项
  • 发邮件，当面汇报，等待审批，均可
  • 仪式感
  • 不是，而是可以换一种形式立项

核心在于一定有一个输出物，一般就是立项文档。一定要有什么东西代表这个东西，仪式感。

分期

  • 怎么分期
  • 为什么要分期
  • 分期规划了就不能调整了么
  • 时间条件，技术条件，目标导向
  • 因为资源有限，竞争激烈
  • 必须可以调整

怎么调整呢：看反馈，看竞品，把更能体现核心产品竞争力的特性提前做出来，其二看数据，把用户更喜欢的特星店做到更好的体验。

RoadMap

用工具来呈现项目计划。

  • 明确RoadMap的排期基本目标
    • 有限资源下，优先选择更重要的事情先做，逐步达到产品目标
  • 需求的合理并行串行
    • 需求之间有关联及递进关系
  • MVP产品理念
    • 最小可行性产品原则，先满足基本需求。

互联网瞬息万变，资源有限，设计可能不完美，需要反馈，先用最高效的方式先满足用户的核心需求

如何使用项目规划

  • 时间管理：把控好开发节奏
  • 预期管理：管理好领导的预期
  • 复盘分析：上线后需要回头检验

每个任务如何规划时长

注意测试的时间，一般要0.5-1倍之间。

产品方案的撰写

一个例子

例子：想要给宝宝身上装个隐形追踪器，让宝宝不丢失。

需求：透过现象看本质，需求本质就是父母希望孩子安全。

分析用户：父母

分析背景：Amber Alter，儿童失踪信息紧急发布平台

分析行业：儿童智能手表，儿童智能手机，便捷式定位器

产品定位：智能定位产品

产品形态：可贴在衣服上的智能定位产品，可与家长通讯设备可连接

预期：一期产品：可续航，无sim卡，便于携带；二期产品：更轻便，能在身体表面，对身体无害

撰写

接下来就是方案撰写，需要满足如下要求

  • 信息加工再展示：适当利用图形化文字，关键文字不可省，使用图形化结构观点。人对图的接收比文字快。
  • 产品目标要清晰
  • 展示期望蓝图

方案陈述技巧

  • 这次要做什么
  • 为什么要这样做
  • 需求是否得到充分分析
  • 需求方案是否得到全面的论证
  • 自信
  • 只要你对方案有信心，就要把信心展示出来。

构建产品结构

概念

产品结构定义：基于前期的需求分析及市场竞品分析等为依据，将各个需求以某种逻辑系统化组织起来所形成的立体结构。

基于该结构，可以顺利的引导用户行为或将各类信息进行顺畅的流转

我们需要做的有关产品结构的构建，就是把原本可能无序的各个需求点以某种结构的方式展示出来。

产品结构类型

常见产品结构：

  1. 线性结构：数组
  2. 层级结构：树
  3. 矩阵结构：十字链表
  4. 自然结构：图

案例层级结构——手机QQ

第一步战略思考

战略

  • 即时通讯软件

  • 年轻人社交工具

  • 生活方式

用户及场景

  • 年轻人，90后
  • 非家长沟通
  • 工作，生活，学习沟通

第二步范围层思考

需求

  • 聊天
  • 图片
  • 视频
  • 传文件
  • ….

任务流程

  • 登录注册
  • 加好友
  • 发起聊天
  • ….

结构层

这个结构就是层级结构，符合人类天生哎分类的习惯，行为效率更高，大多数产品都是用这种结构。

案例线性结构

更适合于讲述故事，比如介绍页。

战略层思考

战略

  • 促进销售成单
  • 提高人均销售额

用户及场景

  • 贪便宜
  • 懒
  • 不方便实体购物的

范围层思考

需求

  • 展现当前商品
  • 提供购买入口
  • 提供促销展示
  • 提供更多商品

任务流程

  • 查看商品
  • 点击购买
  • 查看其它商品
  • 查看评论

案例矩阵结构

比如门户网站，大量信息展示。

自然结构

没有逻辑关系，分类路径不明显，鼓励探索。

真实现状

将各种结构进行综合在一起。

延伸思考

产品结构不单纯是页面解耦股。产品结构=物理结构+逻辑结构。我们需要做的产品结构是这两者的完美结合。

产品结构设计的基本方法

自上而下

自下而上

马蜂窝

寻找共性，然后得到一点。

一般是讲两者结合，自上而下容易忽略细节，自下而上缺乏扩展。可以找到一个中间层，连贯上下。

分类

按照不同标准分类，可以是时间、地点、主题、任务、混合标准如果哟多个维度，可以根据需求的优先级，重要程度，使用场景及频率来划分。

结构化

一些技巧

可以以功能模块作为产品结构

方法有：

  1. 经验法：根据过往经验，比如涉及钱的就一定登录注册，密码等等
  2. 对比法：参考其他产品，列出常用功能
  3. 创造法：根据个人需求，创造新功能

快捷入口

  • 把常用功能放在首页展示
  • 把用户可能需要的功能在相关页面展示
  • 比如微信+，搭配购买

产品结构设计之后的输出物

包含产品结构图：用于展示产品的各个功能模块的立体结构，包含物理结构也有逻辑结构。

用户流程图：描述一用户的视角对一个产品或功能时可进行的操作以及对应的反馈。以用户的视角，在遇到的每一个可以进行的操作是穷尽一切可能的情况，继而给出对应的反馈。

业务流程图

绘制技巧，充分理解业务逻辑，知道熟知任务所涉及到的相关支持部门。
